# README.md Schutzregel (Highest Priority)

## Verboten
- bestehende README.md Abschnitte umschreiben
- Stil/Layout/Struktur verändern
- Überschriften neu ordnen
- vorhandene Inhalte entfernen oder umformatieren

## Erlaubt
- Nur neue Features am Ende hinzufügen
- Neue Abschnitte ausschließlich als Append am Ende

## Konfliktregel
Wenn eine strukturelle Änderung nötig wirken sollte:
- README nicht umbauen
- stattdessen separate Doku-Datei vorschlagen

## Priorität
Höchste Priorität im gesamten Regelset.
